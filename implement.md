# Phase 1 — Project Setup, Landing Page & Tenant Login

## Overview

Set up the Next.js frontend with ShadCN, Tailwind CSS v4, Axios, and next-pwa. Build the **landing page** (global PG/hostel search) and **dynamic tenant login pages** (`/[slug]/login`) that load the organization's own logo and color theme from the API.

No full dashboard modules are wired up in Phase 1 — only the entry flows are built so you can test before Phase 2.

---

## Open Questions

> [!IMPORTANT]
> **Q1 — Org search API**: The server has a platform-level `GET /api/platform/organizations/list` (requires super-admin auth) and a signup/login flow, but there is **no public unauthenticated endpoint to search organizations by name/slug**. Phase 1 needs one to power the landing page search box.
>
> **Options:**
> - A) I add a new `GET /api/public/orgs/search?q=name` route on the server (unauthenticated, returns `name`, `slug`, `logo_url`) — **recommended**.
> - B) You manually seed/hardcode a small JSON file of orgs in the frontend for now and we add the real API later.

> [!IMPORTANT]
> **Q2 — Org branding fields**: The current `organization` model stores `name`, `slug`, `contact_email`, etc. but **no logo URL or brand color**. The login page needs these.
>
> **Options:**
> - A) I add `logo_url`, `brand_color`, `tagline` columns to the Prisma schema (migration needed) — **recommended**.
> - B) Store branding in a separate `OrgBranding` table alongside `org_features`.
> - C) Use a static default for Phase 1 and defer branding to Phase 2.

> [!NOTE]
> **Q3 — Tailwind version**: The project already has Tailwind **v4** (`@tailwindcss/postcss` + `@import "tailwindcss"` style). ShadCN currently has beta support for Tailwind v4. I will use the v4-compatible ShadCN CLI (`shadcn@canary`) — confirm if you want the stable v3 branch instead (requires downgrading Tailwind).

### 3 · Landing Page — `app/page.tsx`

- Clean centered layout: app name/logo, tagline, search input.
- `SearchBox` component: debounced input → calls `GET /api/public/orgs/search?q=` → shows dropdown with org name + logo.
- Clicking a result navigates to `/{slug}/login`.
- Fully keyboard-accessible (Combobox from ShadCN).

---

### 4 · Dynamic Login Page — `app/[slug]/login/page.tsx`

- Server component fetches org by slug from `GET /api/public/orgs/:slug` (returns `name`, `slug`, `logoUrl`, `brandColor`).
- If org not found → 404 page.
- Renders `LoginForm` with org's logo and brand color injected as CSS custom property (`--brand`).
- Layout mirrors the example screenshot: centered card, org logo at top, email + password fields, Login button.
- No sidebar/nav on this page — standalone tenant-branded page.
